# 插件热更新配置

1. 修改配置（开发环境）
    - 打开 WebSite\App_Data\Common.config，在 appSettings 中加入或修改：
```xml
<!-- 开发时启用插件热更新；正式环境请设为 false 以免影响性能 -->
<add key="IsEnablePlugInHotUpdate" value="true" />
```
    - 修改后重启 IIS。

2. 准备临时目录及权限
    - 在 WebSite\TempfilePath 下新建目录 PluginLoaderTempBin。
    - 给予 IIS 进程所属用户（如 IIS_IUSRS 或 Everyone）读写权限。

3. 在插件类上添加热更新特性
```csharp
using Kingdee.BOS.Util;

[HotUpdate]
public class MyPlugin : IMyPlugin
{
     // 插件实现
}
```
    - 编译并部署插件，热更新开启后可在开发过程中替换 DLL 进行验证。

附注：生产环境请务必将 IsEnablePlugInHotUpdate 设为 false 并严格控制目录权限。
